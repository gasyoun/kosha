# Data statement — Zaliznyak declension-class drills (`zaliznyak-drills`)

_Created: 23-07-2026 · Last updated: 23-07-2026_

Data statement for the `zaliznyak-drills` dataset served by the kosha
data-hub. Manifest row:
[data/manifest/datasets.json](https://github.com/gasyoun/kosha/blob/main/data/manifest/datasets.json).
Committed asset:
[`data/zaliznyak/zaliznyak_drills.json`](https://github.com/gasyoun/kosha/blob/main/data/zaliznyak/zaliznyak_drills.json).

## Composition & counts

3,434 multiple-choice items over 332 PWG Zaliznyak declension-class tokens
(`ind·*` indeclinable tokens excluded — nothing to decline). JSON, UTF-8, one
item per row:

| Field | Content |
|---|---|
| `id` | `ZD-####` |
| `type` | `classify` (2,000 items — headword → declension-class token) or `odd-one-out` (1,434 items — 4 headwords, pick the one that inflects differently) |
| `question` / `answer` / `choices` | 4-way MCQ; question renders the headword in Devanāgarī + IAST |
| `index_token` | the Zaliznyak-style class token, e.g. `f·1`, `m·8i` |
| `gender`, `stem_class`, `member_count` | class metadata |
| `tags` | `[type, gender, stem_class]` |

Companion files: `data/zaliznyak/zaliznyak_paradigm_classes.tsv` (slim
committed class index — token/gender/stem_class/member_count/representative,
332 rows), `data/zaliznyak/zaliznyak_drills.tsv` (flat fallback),
`data/zaliznyak/zaliznyak_drills.apkg` (Anki deck, genanki). Web quiz:
[`reading/zaliznyak/drills/`](https://github.com/gasyoun/kosha/blob/main/reading/zaliznyak/drills/index.html).

## Source provenance

Derived in [gasyoun/kosha](https://github.com/gasyoun/kosha)
(`scripts/build_zaliznyak_drills.py`) by reading, at build time only (never
vendored), the PWG Zaliznyak paradigm-class index registered as **E31** in
[SanskritLexicography/FEATURES_INDEX.md](https://github.com/gasyoun/SanskritLexicography/blob/master/FEATURES_INDEX.md):
[`RussianTranslation/src/headword_index.tsv`](https://github.com/gasyoun/SanskritLexicography/blob/master/RussianTranslation/src/headword_index.tsv)
(98,639 PWG headwords → declension-class token) and
[`reverse_paradigm_index.json`](https://github.com/gasyoun/SanskritLexicography/blob/master/RussianTranslation/src/reverse_paradigm_index.json)
(335 class tokens → member headwords), both generated by
[`reverse_index.py`](https://github.com/gasyoun/SanskritLexicography/blob/master/RussianTranslation/src/reverse_index.py)
from the Cologne PWG dictionary (`csl-orig`). Headword transliteration
(SLP1 → Devanāgarī / IAST) via `sanskrit-util` — no new transcoder written.

## Curation rationale

Different asset, repo, and question than the two prior declension-drill
handoffs: [H1296](https://github.com/gasyoun/Uprava/blob/main/handoffs/archive/H1296-Opus_SanskritGrammar_sangram-attested-cell-declension-drills_19.07.26.md)
(SanskritGrammar/sangram) drills corpus-**attested cells** of kosha lemmas
("is this cell attested?"); kosha's own H946 morphology drills
(`morphology-drills` dataset) drill a specific inflected **form** of a
DCS-attested cell. This dataset drills **paradigm-class membership** of a PWG
headword — "which declension class does this word belong to, and which other
words share its paradigm" — with no corpus-attestation join at all; same-class
headwords are the distractor pool. `index_token` grammar is
`<gender>·<section>[+N]`; the `+N` compound-arity suffix does not change the
declension pattern (e.g. `m·1` and `m·1+2` are both `stem_class=a-stem`,
verified empirically against `headword_index.tsv`), so distractor bucketing
keys on `stem_class`, not the raw section string.

## Language variety

Sanskrit (ISO 639-3 `san`) headwords as catalogued in PWG (Böhtlingk-Roth,
Petersburg Dictionary), a 19th-century print dictionary — vocabulary skews
toward śāstra/epic/kāvya registers PWG covers, with the usual pre-1900
lexicographic gaps (no vernacular-adjacent or inscriptional Sanskrit).

## Annotator / process information

Deterministic build (seeded RNG, seed 20260722): class assignment and
member lists are inherited unmodified from `reverse_index.py`'s Zaliznyak
grammar computation (`nominal_grammar.py`, rule-based, no LLM). No
generative content — every question/answer pair is a direct lookup against
the E31 index; disagreements are not possible by construction (the index is
the single source of truth for both question and answer).

## Known biases & limitations

- **PWG-scoped, not corpus-scoped:** a word's declension class here reflects
  PWG's own lexicographic classification, not corpus attestation — unlike
  H946/H1296, there is no claim that any inflected form actually occurs in a
  text.
- **Homonym collisions:** a small number of same-spelling headwords with
  different `hom` indices can render identically in the Devanāgarī+IAST
  drill face; the builder detects and skips any `odd-one-out` group where
  this would produce duplicate on-screen choices (see script comment), so no
  ambiguous item ships, but the affected headword pairs are simply dropped
  from that group rather than resolved.
- **Thin gender buckets:** `mf`/`fn`/`mn` genders have very few tokens (4–7),
  so `classify` distractors for those items fall back to cross-gender
  same-question tokens of a different `stem_class` rather than same-gender
  ones (see `pick_distractor_tokens` fallback in the builder) — an honest
  relaxation, not a silent gap.
- **Deck coverage is sampled, not exhaustive:** items are capped per token
  (15 classify / 8 odd-one-out groups) so no single high-frequency class
  (e.g. `m·1+2`, 12,681 members) dominates the deck; the full class→member
  mapping remains in `reverse_paradigm_index.json` upstream for any consumer
  needing exhaustive coverage.

## Intended use / known misuse

**For:** active-recall practice on PWG's declension-class taxonomy;
Anki-deck study; a companion surface to the corpus-attestation-based drills
(H946, H1296). **Misuse:** treating an item's answer as evidence a form is
corpus-attested (it is not — see limitations); re-deriving the class index
from PWG XML instead of reading E31's committed derived files.

Rebuilt by re-running `scripts/build_zaliznyak_drills.py` whenever the
SanskritLexicography E31 index is regenerated (`reverse_index.py --build`
upstream); no independent kosha-side regeneration trigger. Sunset: superseded
in place on rebuild, no versioned snapshot retained beyond the committed
current file.
